# task-stats tests

This folder holds the automated test stack for `task-stats`.

The idea is simple:

- unit tests check the pure logic
- integration tests hit the Windows-backed seams
- visual e2e tests render deterministic screenshots and can ask a cheap vision model to judge them

## Test layers

### Unit

Project: `tests\unit\TaskStats.UnitTests.csproj`

Covers things that should stay fast and deterministic:

- `CircularBuffer`
- settings serialize / parse round-trips
- layout width math
- hit-testing
- formatting helpers

Run it with:

```bat
run-unit-tests.bat
```

### Integration

Project: `tests\integration\TaskStats.IntegrationTests.csproj`

Covers real Windows-facing behaviour that still benefits from automation:

- settings persistence to disk
- startup registration
- live metric sampling contracts

Run it with:

```bat
run-integration-tests.bat
```

### Visual e2e

The visual stack has two parts:

1. `tests\visual\TaskStats.VisualHarness.csproj`
   This launches `OverlayForm` with fake metrics, fixed settings, and deterministic bounds so the screenshots are stable.

2. `tests\e2e\run-e2e.ps1`
   This builds the app, captures screenshots into `tests\e2e\artifacts`, checks that the artifacts exist, then optionally asks OpenRouter to judge the screenshots.

Run it with:

```bat
run-e2e-tests.bat
```

If you want the local deterministic checks only:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tests\e2e\run-e2e.ps1 -SkipAI
```

## OpenRouter config

The AI screenshot judge reads secrets from the repo-root `.env`.

Relevant keys:

- `OPENROUTER_API_KEY`
- `TASK_STATS_VISION_MODEL`

Default model:

```text
google/gemini-2.5-flash-lite
```

That keeps the visual pass cheap enough to run when you actually need it.

## Artifacts

`tests\e2e\artifacts\`

This folder is regenerated by the visual tests. It contains:

- `aggregate.png`
- `percore.png`
- matching `.json` manifests
- optional `*-screen.png` captures of the real desktop region for debugging

The AI judge should evaluate the deterministic rendered images, not the desktop debug captures.

## When to run what

- changed pure logic, run `run-unit-tests.bat`
- changed persistence, startup, metrics sampling, or other Windows-facing behaviour, run `run-integration-tests.bat`
- changed rendering, layout, labels, colours, or screenshot expectations, run `run-e2e-tests.bat`
- changed anything substantial in `task-stats`, run `run-tests.bat` and then decide whether the visual pass is also needed

## TDD note

For non-trivial `task-stats` changes, add or update the relevant test first.

If the code has no clean seam yet, extract the seam first, then add the test, then change the behaviour.
